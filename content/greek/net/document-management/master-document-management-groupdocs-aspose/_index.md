# Δημιουργία ευρετηρίου Aspose Search με το GroupDocs Redaction για .NET

Δημιουργήστε αποδοτικά αρχεία **create Aspose Search index** και συνδυάστε τα με το GroupDocs Redaction για να δημιουργήσετε μια ισχυρή λύση διαχείρισης εγγράφων για εφαρμογές .NET. Είτε είστε επαγγελματίας IT που θέλει να βελτιστοποιήσει τεράστιες συλλογές εγγράφων είτε προγραμματιστής που προσθέτει δυνατότητες αναζητήσιμης διαγραφής, αυτός ο οδηγός σας καθοδηγεί βήμα προς βήμα — από τη ρύθμιση του περιβάλλοντος μέχρι την ενσωμάτωση των δύο προϊόντων σε μια παραγωγική ροή εργασίας.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει το “create Aspose Search index”;** Σημαίνει τη δημιουργία ενός αναζητήσιμου αποθετηρίου που το Aspose Search μπορεί να ερωτήσει άμεσα.  
- **Ποια έκδοση .NET απαιτείται;** .NET Framework 4.7.2 or later, or .NET Core 3.1 +.  
- **Χρειάζομαι άδεια;** Ναι—χρησιμοποιήστε μια δωρεάν δοκιμή ή προσωρινή άδεια για αξιολόγηση, στη συνέχεια αγοράστε για παραγωγή.  
- **Μπορεί το GroupDocs Redaction να λειτουργήσει με το ευρετήριο;** Απόλυτα· μπορείτε να διαγράψετε έγγραφα πριν ή μετά την ευρετηρίασή τους.  
- **Πόσες μορφές υποστηρίζονται;** Το Aspose Search διαχειρίζεται πάνω από 30 τύπους αρχείων, και το GroupDocs Redaction υποστηρίζει πάνω από 150 μορφές εγγράφων.

## Τι είναι το “create Aspose Search index”;
Ένα ευρετήριο Aspose Search είναι μια μόνιμη δομή αποθήκευσης που εξάγει κείμενο από υποστηριζόμενα αρχεία και το οργανώνει σε αντιστροφή λιστών, επιτρέποντας ερωτήματα βασισμένα σε λέξεις‑κλειδιά να επιστρέφουν αποτελέσματα σε χιλιοστά του δευτερολέπτου. Με τη δημιουργία αυτού του ευρετηρίου μετατρέπετε ακατέργαστα έγγραφα σε μια αναζητήσιμη βάση γνώσης που μπορεί να ερωτηθεί αποδοτικά ακόμη και καθώς η συλλογή μεγαλώνει.

## Γιατί να χρησιμοποιήσετε το GroupDocs Redaction μαζί με το Aspose Search;
Το GroupDocs Redaction παρέχει αυτοματοποιημένη διαγραφή ευαίσθητων πληροφοριών, ενώ το Aspose Search προσφέρει εξαιρετικά γρήγορη αναζήτηση πλήρους κειμένου. Μαζί σας επιτρέπουν να **securely index** και **search** εκατομμύρια έγγραφα χωρίς να εκθέτετε εμπιστευτικά δεδομένα. Το Aspose Search μπορεί να επεξεργαστεί έως **1 million documents** ανά αποθετήριο και υποστηρίζει **30+ input formats**, ενώ το GroupDocs Redaction μπορεί να διαχειριστεί **150+ document types** σε μια ενιαία λειτουργία.

## Οδηγός Υλοποίησης
Παρακάτω θα βρείτε έναν οδηγό βήμα‑βήμα που δείχνει πώς να **create Aspose Search index** και να το συνδέσετε με το GroupDocs Redaction.

### Πώς να δημιουργήσετε το “create Aspose Search index”;
Φορτώστε το Aspose.Search SDK, επισημάνετε έναν φάκελο και καλέστε το `CreateRepository`. Το CreateRepository είναι μια στατική μέθοδος που αρχικοποιεί ένα νέο αποθετήριο στη συγκεκριμένη διαδρομή, εκχωρώντας τα απαραίτητα αρχεία και μεταδεδομένα. Αυτή η ενιαία κλήση δημιουργεί τη δομή του ευρετηρίου στο δίσκο και το προετοιμάζει για την εισαγωγή εγγράφων, επιτρέποντας στις επόμενες λειτουργίες ευρετηρίου να εκτελούνται αποδοτικά.

#### Βήμα 1: Ορισμός Διαδρομών Φακέλου Ευρετηρίου
```csharp
string indexFolder1 = @"YOUR_DOCUMENT_DIRECTORY/UsingIndexRepository/Index1";
string indexFolder2 = @"YOUR_DOCUMENT_DIRECTORY/UsingIndexRepository/Index2";
```  

#### Βήμα 2: Δημιουργία Αντικειμένου `IndexRepository`
`IndexRepository` είναι η κεντρική κλάση του Aspose Search που αντιπροσωπεύει μια συλλογή ενός ή περισσότερων ευρετηρίων στο σύστημα αρχείων.

```csharp
using GroupDocs.Search;
// Instantiate the repository
IndexRepository indexRepository = new IndexRepository();
```  

### Πώς να προσθέσετε ευρετήρια στο αποθετήριο;
Η προσθήκη ευρετηρίων σας επιτρέπει να διαχωρίζετε έγγραφα ανά τμήμα, έργο ή οποιαδήποτε λογική ομαδοποίηση, ενώ το αποθετήριο παρακολουθεί γεγονότα προόδου για άμεση ανατροφοδότηση. Ένα αντικείμενο Index ενσωματώνει τα δικά του αντιστροφή αρχεία και ρυθμίσεις, επιτρέποντάς σας να απομονώσετε περιοχές αναζήτησης και να εφαρμόσετε διαφορετικούς αναλυτές ανά ομάδα. Το αποθετήριο ενεργοποιεί γεγονότα προόδου ώστε να μπορείτε να εμφανίζετε ενημερώσεις κατάστασης ή να εκκινείτε ενέργειες καθώς η ευρετηρίαση προχωρά.

#### Εγγραφή στο Γεγονός Προόδου Λειτουργίας
```csharp
indexRepository.Events.OperationProgressChanged += (sender, args) =>
{
    // Implement event logic here
};
```  

#### Προσθήκη Ευρετηρίων στο Αποθετήριο
Δημιουργήστε ή φορτώστε ευρετήρια και προσθέστε τα:

```csharp
Index index1 = new Index(indexFolder1);
indexRepository.AddToRepository(index1);

Index index2 = new Index(indexFolder2);
indexRepository.AddToRepository(index2);
```  

### Πώς να προσθέσετε έγγραφα στα ευρετήρια;
Συμπληρώστε κάθε ευρετήριο με τα αρχεία που θέλετε να είναι αναζητήσιμα. Το API εξάγει αυτόματα κείμενο από υποστηριζόμενες μορφές. Χρησιμοποιήστε τη μέθοδο `AddDocument` για να δώσετε μια διαδρομή αρχείου· το `AddDocument` εξάγει το περιεχόμενο του εγγράφου, δημιουργεί τα απαραίτητα tokens και τα αποθηκεύει στο επιλεγμένο ευρετήριο. Αυτή η διαδικασία εξασφαλίζει ότι όλα τα αναζητήσιμα πεδία είναι ευρετηριασμένα και έτοιμα για ερωτήματα.

#### Ορισμός Διαδρομών Φακέλου Εγγράφων
```csharp
string documentFolder1 = @"YOUR_DOCUMENT_DIRECTORY/UsingIndexRepository/Documents1";
string documentFolder2 = @"YOUR_DOCUMENT_DIRECTORY/UsingIndexRepository/Documents2";
```  

#### Προσθήκη Εγγράφων σε Συγκεκριμένα Ευρετήρια
```csharp
index1.Add(documentFolder1);
index2.Add(documentFolder2);
```  

### Πώς να ενημερώσετε τα ευρετήρια στο αποθετήριο;
Διατηρήστε τα αποτελέσματα αναζήτησης ενημερωμένα καλώντας τη μέθοδο ενημέρωσης κάθε φορά που αλλάζουν τα αρχεία προέλευσης. Η μέθοδος `Update` επεξεργάζεται ξανά τα τροποποιημένα ή νεοπροστέθηκαν αρχεία, ξαναδημιουργεί τις επηρεαζόμενες αντιστροφή λίστες και συγχρονίζει τα μεταδεδομένα του αποθετηρίου. Η τακτική εκτέλεση αυτής της λειτουργίας εξασφαλίζει ότι τα ερωτήματα αντανακλούν το πιο πρόσφατο περιεχόμενο των εγγράφων χωρίς να απαιτείται πλήρης επαναδημιουργία.

```csharp
// Update all indexes
indexRepository.Update();
```  

### Πώς να κάνετε αναζήτηση στο αποθετήριο;
Εκτελέστε ένα ερώτημα που καλύπτει όλα τα ευρετήρια, επιστρέφοντας αποτελέσματα με επισημασμένα αποσπάσματα. Η μέθοδος `Search` δέχεται μια συμβολοσειρά ερωτήματος, την επεξεργάζεται σε κάθε ευρετήριο και επιστρέφει μια συλλογή αντικειμένων SearchResult που περιλαμβάνουν αναφορές εγγράφων, βαθμολογίες συνάφειας και επισημασμένα αποσπάσματα. Μπορείτε να βελτιώσετε περαιτέρω τα αποτελέσματα χρησιμοποιώντας φίλτρα, ταξινόμηση ή σελιδοποίηση ώστε να ταιριάζουν στις ανάγκες της εφαρμογής.

#### Ορισμός Ερωτήματος Αναζήτησης και Εκτέλεση Αναζήτησης
```csharp
using GroupDocs.Search.Results;
string query = "decisively";
SearchResult result = indexRepository.Search(query);
```

- **Corporate Knowledge Bases** – Αναζητήστε με ασφάλεια εσωτερικά εγχειρίδια ενώ κρύβετε αυτόματα ιδιόκτητες πληροφορίες.

## Σκέψεις Απόδοσης
- Χρησιμοποιήστε επαυξητική ευρετηρίαση για να αποφύγετε την επαναδημιουργία μεγάλων ευρετηρίων από την αρχή.  
- Προγραμματίστε τακτικές ενημερώσεις του αποθετηρίου κατά τις ώρες χαμηλής ζήτησης.  
- Παρακολουθείτε CPU και μνήμη· το Aspose Search επεξεργάζεται έως **500 MB/s** σε έναν τυπικό διακομιστή 8‑πυρήνων.

## Συνηθισμένα Προβλήματα και Λύσεις
- **Index build fails due to file permissions** – Βεβαιωθείτε ότι ο λογαριασμός υπηρεσίας έχει πρόσβαση ανάγνωσης/εγγραφής στο φάκελο του ευρετηρίου.  
- **Redaction does not apply before search** – Καλέστε `Redactor.Redact()` πριν προσθέσετε το έγγραφο στο ευρετήριο.  
- **Search returns stale results** – Εκτελέστε `indexRepository.Update()` μετά από οποιεσδήποτε μαζικές αλλαγές εγγράφων.
