# Αναζήτηση Πλήρους Κειμένου Java με GroupDocs.Search

## Εισαγωγή
Αν παλεύετε με **full text search java** σε αμέτρητα αρχεία, δεν είστε μόνοι. Η χειροκίνητη σάρωση PDF, Word ή υπολογιστικών φύλλων γρήγορα γίνεται εμπόδιο. Ευτυχώς, το GroupDocs.Search για Java σας επιτρέπει να αυτοματοποιήσετε αυτή τη διαδικασία, παρέχοντας γρήγορα, ακριβή αποτελέσματα για οποιονδήποτε τύπο εγγράφου. Σε αυτό το tutorial θα περάσουμε βήμα‑βήμα από τη ρύθμιση της βιβλιοθήκης μέχρι την προσθήκη εγγράφων στο ευρετήριο, τη δημιουργία δηλώσεων boolean query java και την **βελτιστοποίηση της απόδοσης αναζήτησης**. Στο τέλος, θα έχετε μια πλήρη, έτοιμη για παραγωγή υλοποίηση full text search java στην εφαρμογή σας.

## Γρήγορες Απαντήσεις
- **Τι είναι full text search java;** Μια τεχνική που δημιουργεί ευρετήριο του ακατέργαστου κειμένου των εγγράφων ώστε να μπορείτε να ερωτήσετε οποιαδήποτε λέξη ή φράση άμεσα.  
- **Ποια βιβλιοθήκη υποστηρίζει πολλαπλές μορφές;** Το GroupDocs.Search για Java διαχειρίζεται PDF, DOCX, XLSX και πολλά άλλα.  
- **Πώς προσθέτω έγγραφα στο ευρετήριο;** Χρησιμοποιήστε τη μέθοδο `index.add()` με διαδρομή ή ένα προσαρμοσμένο `DocumentFilter`.  
- **Μπορώ να εκτελώ ερωτήματα Boolean;** Ναι—συνδυάστε όρους με AND, OR, NOT για ακριβή αποτελέσματα.  
- **Πώς βελτιώνω την απόδοση;** Ενημερώνετε τακτικά το ευρετήριο, ενεργοποιήστε την προσωρινή αποθήκευση (caching) και ενεργοποιήστε την φωνητική αναζήτηση μόνο όταν χρειάζεται.

## Τι είναι η Full Text Search Java;
Η full text search java είναι η διαδικασία σάρωσης ολόκληρου του κειμενικού περιεχομένου των εγγράφων, αποθήκευσης του σε αποδοτικό ευρετήριο και στη συνέχεια επιτρέπει γρήγορα ερωτήματα λέξεων-κλειδιών ή φράσεων. Σε αντίθεση με τις απλές αναζητήσεις με βάση το όνομα αρχείου, κοιτάζει μέσα στα αρχεία, καθιστώντας την ιδανική για συστήματα διαχείρισης εγγράφων, πύλες υποστήριξης και οποιοδήποτε σενάριο όπου οι χρήστες χρειάζονται γρήγορη εντόπιση πληροφοριών.

## Γιατί να Χρησιμοποιήσετε το GroupDocs.Search για Java;
- **Υποστήριξη πολλαπλών μορφών** – Word, PDF, Excel, PowerPoint και άλλα.  
- **Κλιμακωτή δημιουργία ευρετηρίου** – Διαχειρίζεται εκατομμύρια αρχεία με μικρό αποτύπωμα μνήμης.  
- **Προηγμένη γλώσσα ερωτημάτων** – Boolean, fuzzy και phonetic αναζητήσεις έτοιμες για χρήση.  
- **Εύκολη ενσωμάτωση** – Απλή εξάρτηση Maven και απλό API.

### Άμεση Λήψη (αν προτιμάτε χειροκίνητη εγκατάσταση)

## Οδηγός Υλοποίησης

### Προσθήκη Εγγράφων στο Ευρετήριο
**Γιατί είναι σημαντικό:** Χωρίς περιεχόμενο στο ευρετήριο δεν υπάρχουν αποτελέσματα αναζήτησης. Παρακάτω δείχνουμε πώς να προσθέσετε ολόκληρους φακέλους ή να φιλτράρετε συγκεκριμένους τύπους αρχείων.

#### Βήμα 1: Δημιουργία Ευρετηρίου
```java
Index index = new Index("C:\\MyIndex");
```

#### Βήμα 2: Προσθήκη Εγγράφων (add documents to index)
Μπορείτε να ευρετηριάσετε τα πάντα σε έναν φάκελο ή να περιορίσετε σε συγκεκριμένες επεκτάσεις:

```java
index.add("C:\\Documents\\*.*"); // Adds all documents from the specified directory
// For specific file types, use:
index.add("C:\\Reports", new DocumentFilter() {
    @Override
    public boolean accept(String fileName) {
        return fileName.endsWith(".pdf") || fileName.endsWith(".docx");
    }
});
```

> **Επεξήγηση:**  
> - `Index` αντιπροσωπεύει τη βάση δεδομένων αναζήτησης.  
> - `add()` εισάγει αρχεία· το σύμβολο μπαλαντέρ `*.*` παίρνει όλα τα αρχεία, ενώ το `DocumentFilter` σας επιτρέπει να ρυθμίσετε λεπτομερώς το βήμα **add documents to index**.

### Εκτέλεση Αναζήτησης (search documents java)
Τώρα που το ευρετήριο περιέχει δεδομένα, μπορείτε να το ερωτήσετε.

#### Βήμα 1: Δημιουργία Ερωτήματος
```java
String query = "GroupDocs";
```

#### Βήμα 2: Εκτέλεση της Αναζήτησης
```java
SearchResult result = index.search(query);
System.out.println("Documents found: " + result.getDocumentCount());
```

> **Επεξήγηση:**  
> - `search()` εκτελεί το ερώτημα στο ευρετήριο.  
> - `getDocumentCount()` επιστρέφει πόσα έγγραφα ταιριάζουν—χρήσιμο για γρήγορους ελέγχους.

### Προχωρημένες Τεχνικές Ερωτημάτων (boolean query java)
Για ακριβή έλεγχο, συνδυάστε όρους με λογική Boolean.

#### Boolean Queries
```java
String booleanQuery = "GroupDocs AND Java";
SearchResult booleanResult = index.search(booleanQuery);
```

#### Phonetic Searches (optional for fuzzy matching)
```java
index.getSettings().setPhoneticSearch(true);
```

> **Πότε να το χρησιμοποιήσετε:** Ενεργοποιήστε τη φωνητική αναζήτηση μόνο αν οι χρήστες συχνά γράφουν λανθασμένα τους όρους· διαφορετικά, κρατήστε την απενεργοποιημένη για **βελτιστοποίηση της απόδοσης αναζήτησης**.

## Πρακτικές Εφαρμογές
Το GroupDocs.Search ξεχωρίζει σε πραγματικά σενάρια:

1. **Συστήματα Διαχείρισης Περιεχομένου** – Παρέχουν άμεση full‑text αναζήτηση σε άρθρα, PDF και πολυμέσα.  
2. **Πύλες Εξυπηρέτησης Πελατών** – Οι πράκτορες εντοπίζουν σχετικές οδηγίες ή πολιτικές σε δευτερόλεπτα.  
3. **Εταιρικά Αποθετήρια Εγγράφων** – Αναζητούν συμβόλαια, εκθέσεις και έγγραφα συμμόρφωσης χωρίς μεταφορά δεδομένων σε ξεχωριστή βάση.

## Σκέψεις για την Απόδοση
### Βελτιστοποίηση της Απόδοσης Αναζήτησης
- **Incremental Indexing:** Προσθέστε ή ενημερώστε μόνο τα αλλαγμένα αρχεία αντί να ξαναδημιουργείτε ολόκληρο το ευρετήριο.  
- **Caching:** Διατηρήστε συχνά χρησιμοποιούμενα αποτελέσματα ερωτημάτων στη μνήμη.  
- **Παρακολούθηση Πόρων:** Ρυθμίστε το heap της JVM (`-Xmx2g` κ.λπ.) ανάλογα με το μέγεθος του ευρετηρίου.

### Οδηγίες Χρήσης Πόρων
- Κρατήστε το φάκελο ευρετηρίου σε γρήγορο δίσκο.  
- Παρακολουθείτε CPU και μνήμη κατά τη μαζική ευρετηρίαση· οι παρτίδες μπορούν να ρυθμιστούν ώστε να αποφεύγουν αιχμές.

### Καλές Πρακτικές για Διαχείριση Μνήμης Java
- Χρησιμοποιήστε `try-with-resources` όταν εργάζεστε με streams.  
- Απενεργοποιήστε (nullify) μεγάλα αντικείμενα μετά τη χρήση για να βοηθήσετε τη συλλογή απορριμμάτων.
