# Προσθήκη Πολλαπλών Ψευδώνυμων και Προσθήκη Εγγράφων στο Ευρετήριο στο GroupDocs.Search Java: Ένας Πλήρης Οδηγός

Στη σημερινή εποχή που βασίζεται στα δεδομένα, η δυνατότητα **προσθήκη πολλαπλών ψευδώνυμων** ενώ **προσθέτετε έγγραφα στο ευρετήριο** δίνει στη λύση αναζήτησής σας ένα σαφές πλεονέκτημα απόδοσης. Είτε ευρετοποιείτε χιλιάδες συμβάσεις, καταλόγους προϊόντων ή ερευνητικές εργασίες, το GroupDocs.Search for Java σας επιτρέπει να **δημιουργήσετε δομές αναζητήσιμου ευρετηρίου** και να ρυθμίσετε ακριβώς τις ερωτήσεις με λεξικά ψευδώνυμων — όλα ενώ η υλοποίηση παραμένει απλή και γρήγορη.

## Γρήγορες Απαντήσεις
- **Ποιο είναι το πρώτο βήμα για να ξεκινήσετε να χρησιμοποιείτε το GroupDocs.Search;** Προσθέστε την εξάρτηση Maven και αρχικοποιήστε ένα αντικείμενο `Index`.  
- **Πώς προσθέτω έγγραφα στο ευρετήριο;** Καλέστε `index.add("<folder_path>")` με το φάκελο που περιέχει τα αρχεία σας.  
- **Μπορώ να δημιουργήσω ψευδώνυμα για σύνθετες ερωτήσεις;** Ναι — χρησιμοποιήστε το λεξικό ψευδώνυμων για να αντιστοιχίσετε σύντομες ενδείξεις σε πλήρεις εκφράσεις ερωτήματος, και μπορείτε επίσης να **προσθέσετε πολλαπλά ψευδώνυμα** μαζικά.  
- **Μπορεί να γίνει εξαγωγή και εισαγωγή λεξικών ψευδώνυμων;** Απόλυτα — χρησιμοποιήστε τις μεθόδους `exportDictionary` και `importDictionary`.  
- **Ποια έκδοση του GroupDocs.Search απαιτείται;** Έκδοση 25.4 ή νεότερη (το tutorial χρησιμοποιεί την 25.4).  

## Τι σημαίνει “προσθήκη εγγράφων στο ευρετήριο”;
Η προσθήκη εγγράφων σε ένα ευρετήριο σημαίνει την τροφοδοσία ακατέργαστων αρχείων (PDF, DOCX, TXT κ.λπ.) στο GroupDocs.Search ώστε η βιβλιοθήκη να μπορεί να αναλύσει το περιεχόμενό τους και να δημιουργήσει ένα **αναζητήσιμο ευρετήριο**. Μόλις γίνει η ευρετοποίηση, μπορείτε να εκτελείτε γρήγορες, πλήρεις ερωτήσεις κειμένου σε όλα αυτά τα έγγραφα.

## Γιατί να Διαχειρίζεστε Ψευδώνυμα;
Τα ψευδώνυμα σας επιτρέπουν να αντικαθιστάτε μεγάλα, επαναλαμβανόμενα τμήματα ερωτήματος με σύντομες, εύκολα θυμητές ενδείξεις (π.χ., `@t` → `(gravida OR promotion)`). Αυτό όχι μόνο συντομεύει τις συμβολοσειρές αναζήτησής σας, αλλά επίσης βελτιώνει την αναγνωσιμότητα, τη συντηρησιμότητα και **βελτιστοποιεί την απόδοση της αναζήτησης**, ειδικά όταν τα ερωτήματα γίνονται σύνθετα.

## Πώς να προσθέσετε πολλαπλά ψευδώνυμα;
Το GroupDocs.Search παρέχει μια βολική μέθοδο `addRange` που σας επιτρέπει να εισάγετε πολλαπλά ζεύγη ψευδώνυμο‑αντικατάστασης ταυτόχρονα. Αυτή η μαζική λειτουργία μειώνει το κόστος σε σύγκριση με την προσθήκη κάθε ψευδώνυμου ξεχωριστά.

## Προαπαιτούμενα

- **GroupDocs.Search for Java** ≥ 25.4.  
- **JDK** (οποιαδήποτε πρόσφατη έκδοση, π.χ., 11+).  
- Ένα IDE όπως το **IntelliJ IDEA** ή το **Eclipse**.  
- Βασικές γνώσεις Java και Maven.  

## Ρύθμιση του GroupDocs.Search για Java

### Χρήση Maven
Προσθέστε το αποθετήριο και την εξάρτηση στο `pom.xml` σας:

```xml
<repositories>
   <repository>
      <id>repository.groupdocs.com</id>
      <name>GroupDocs Repository</name>
      <url>https://releases.groupdocs.com/search/java/</url>
   </repository>

<dependencies>
   <dependency>
      <groupId>com.groupdocs</groupId>
      <artifactId>groupdocs-search</artifactId>
      <version>25.4</version>

</dependencies>
```

[GroupDocs.Search for Java releases](https://releases.groupdocs.com/search/java/).

import com.groupdocs.search.Index;

## Οδηγός Υλοποίησης

Παρακάτω υπάρχει ένας πλήρης οδηγός για κάθε λειτουργία. Διαβάστε πρώτα τις εξηγήσεις, στη συνέχεια αντιγράψτε το αντίστοιχο τμήμα κώδικα.

### Δημιουργία ή Άνοιγμα ενός Ευρετηρίου

**Πώς να προσθέσετε έγγραφα στο ευρετήριο – πρώτα χρειάζεστε ένα ενεργό αντικείμενο Index.**

#### Βήμα 1: Εισαγωγή της κλάσης Index
```java
import com.groupdocs.search.Index;
```

#### Βήμα 2: Ορίστε πού θα αποθηκευτούν τα αρχεία του ευρετηρίου
```java
String indexFolder = "YOUR_DOCUMENT_DIRECTORY/Indexes/Index";
```

#### Βήμα 3: Δημιουργήστε νέο ευρετήριο ή ανοίξτε ένα υπάρχον
```java
Index index = new Index(indexFolder);
```

### Προσθήκη Εγγράφων σε ένα Ευρετήριο

Τώρα που υπάρχει το ευρετήριο, ας **προσθέσουμε έγγραφα στο ευρετήριο**.

#### Βήμα 1: Καθορίστε τον φάκελο προέλευσης
```java
String documentsFolder = "YOUR_DOCUMENT_DIRECTORY/Documents";
```

#### Βήμα 2: Προσθέστε κάθε υποστηριζόμενο αρχείο από αυτόν το φάκελο
```java
index.add(documentsFolder);
```

> **Συμβουλή:** Εκτελέστε αυτό το βήμα κάθε φορά που φτάνουν νέα αρχεία. Το GroupDocs.Search θα ευρετοποιήσει μόνο το νέο περιεχόμενο, αφήνοντας τα υπάρχοντα στοιχεία άθικτα. Αυτό είναι η ουσία του **incremental indexing java**.

### Διαχείριση Λεξικού Ψευδώνυμων

Τα ψευδώνυμα σας επιτρέπουν να αντιστοιχίζετε σύντομες ενδείξεις σε σύνθετες συμβολοσειρές ερωτημάτων. Θα καλύψουμε την εκκαθάριση παλαιών καταχωρήσεων, την προσθήκη μεμονωμένων ψευδώνυμων και την **προσθήκη πολλαπλών ψευδώνυμων** μαζικά.

#### Εκκαθάριση Υπαρχόντων Ψευδώνυμων
```java
if (index.getDictionaries().getAliasDictionary().getCount() > 0) {
    index.getDictionaries().getAliasDictionary().clear();
}
```

#### Προσθήκη Μεμονωμένων Ψευδώνυμων
```java
index.getDictionaries().getAliasDictionary().add("t", "(gravida OR promotion)");
index.getDictionaries().getAliasDictionary().add("e", "(viverra OR farther)");
```

#### Προσθήκη Πολλαπλών Ψευδώνυμων
```java
AliasReplacementPair[] pairs = new AliasReplacementPair[] {
    new AliasReplacementPair("d", "daterange(2017-01-01 ~~ 2019-12-31)"),
    new AliasReplacementPair("n", "(400 ~~ 4000)")
};
index.getDictionaries().getAliasDictionary().addRange(pairs);
```

### Ερώτηση Αντικαταστάσεων Ψευδώνυμων

Μπορείτε να ανακτήσετε το πλήρες κείμενο για οποιοδήποτε ψευδώνυμο έχετε ορίσει:

```java
if (index.getDictionaries().getAliasDictionary().contains("e")) {
    String replacement = index.getDictionaries().getAliasDictionary().getText("e");
}
```

### Εξαγωγή και Εισαγωγή Λεξικού Ψευδώνυμων

Η εξαγωγή είναι χρήσιμη για εφεδρικά αντίγραφα ή κοινή χρήση μεταξύ περιβαλλόντων.

#### Εξαγωγή Ψευδώνυμων
```java
String fileName = "YOUR_OUTPUT_DIRECTORY/Aliases.dat";
index.getDictionaries().getAliasDictionary().exportDictionary(fileName);
```

#### Εισαγωγή Ψευδώνυμων
```java
index.getDictionaries().getAliasDictionary().importDictionary(fileName);
```

### Αναζήτηση Χρησιμοποιώντας Ερωτήματα Ψευδώνυμων

Με τα ψευδώνυμα σε θέση, οι συμβολοσειρές αναζήτησής σας γίνονται πολύ πιο καθαρές:

```java
String query = "@t OR @e";
SearchResult result = index.search(query);
```

Το σύμβολο `@` λέει στο GroupDocs.Search να αντικαταστήσει την ένδειξη με την πλήρη έκφρασή της πριν εκτελεστεί η αναζήτηση.

## Πρακτικές Εφαρμογές

| Σενάριο | Πώς Βοηθούν τα Ψευδώνυμα |
|----------|-------------------|
| **Διαχείριση Νομικών Εγγράφων** | Αντιστοιχίστε αριθμούς υποθέσεων (`@case123`) σε σύνθετες λογικές εκφράσεις, επιταχύνοντας την ανάκτηση. |
| **Αναζήτηση Προϊόντων στο E‑commerce** | Αντικαταστήστε κοινά συνδυασμένα χαρακτηριστικά (`@sale`) με `(discounted OR clearance)`. |
| **Βάσεις Δεδομένων Έρευνας** | Χρησιμοποιήστε το `@year2020` για να επεκτείνετε σε φίλτρο εύρους ημερομηνιών σε πολλά άρθρα. |

## Σκέψεις Απόδοσης

- **Incremental Indexing:** Προσθέστε μόνο νέα ή τροποποιημένα αρχεία· αποφύγετε την πλήρη επανευρετοποίηση.  
- **JVM Tuning:** Κατανείμετε επαρκή μνήμη heap (`-Xmx4g` για μεγάλα σώματα δεδομένων).  
- **Batch Alias Updates:** Χρησιμοποιήστε το `addRange` για να εισάγετε πολλά ψευδώνυμα ταυτόχρονα, μειώνοντας το κόστος.  
- **Optimize Search Performance:** Διατηρήστε το λεξικό ψευδώνυμων ελαφρύ και επαναχρησιμοποιήστε τις ενδείξεις σοφά για να ελαχιστοποιήσετε το χρόνο ανάλυσης ερωτημάτων.  

## Συχνά Προβλήματα και Λύσεις

| Πρόβλημα | Λύση |
|-------|----------|
| Τα νέα αρχεία δεν είναι αναζητήσιμα | Εκτελέστε ξανά `index.add(newFolder)`· το GroupDocs.Search ευρετοποιεί μόνο τα αόρατα αρχεία. |
| Το ψευδώνυμο επιστρέφει κενό αποτέλεσμα | Επαληθεύστε ότι το κλειδί ψευδώνυμου (`@`) είναι σωστά προθεματισμένο και ότι το λεξικό περιέχει την ένδειξη. |
| Υψηλή χρήση μνήμης κατά τη μαζική ευρετοποίηση | Αυξήστε τη μνήμη heap του JVM (`-Xmx`) και σκεφτείτε να ευρετοποιήσετε σε μικρότερα παρτίδες. |

Α: Απόλυτα — απλώς καλέστε `add` με το ίδιο κλειδί· η βιβλιοθήκη θα αντικαταστήσει την προηγούμενη τιμή.

**Ε: Τι πρέπει να κάνω αν η αναζήτησή μου επιστρέφει μη αναμενόμενα αποτελέσματα;**  
Α: Επαληθεύστε ότι οι ορισμοί των ψευδώνυμων είναι σωστοί, ξαναευρετοποιήστε τυχόν νέα έγγραφα και ελέγξτε τις ρυθμίσεις του αναλυτή για προβλήματα τοκοποίησης.
